# ConvertUnlimited Language Style

## Purpose

ConvertUnlimited should read like a technical utility site: precise, formal, and operational. Copy should help users choose a tool, understand the processing boundary, complete the task, and verify privacy-sensitive claims.

The writing-insights research is useful for avoiding AI-shaped prose, but ConvertUnlimited should not become casual, anecdotal, or blog-like. Use the research as an anti-pattern checklist, then apply a professional tools-site register.

## Voice

- Use formal, direct technical language.
- Prefer clear nouns over marketing adjectives.
- Explain browser behavior, input/output scope, limits, and verification steps.
- Keep privacy language scoped to the specific workflow and build.
- Use short paragraphs, but do not make the page sound conversational.

## Preferred Patterns

- "Selected file contents are processed locally in your browser."
- "ConvertUnlimited does not provide a server-side upload endpoint for this processing flow."
- "The public site may load ads and analytics."
- "Use the privacy build for privacy-sensitive workflows."
- "Output quality depends on browser encoders, source format, and selected settings."
- "Large files are limited by device memory, CPU, and browser support."

## Avoid

- Broad labels such as "privacy-first" on public pages.
- Absolute privacy claims such as "no one can see your files" or "supported processing runs in your browser."
- Casual promises such as "just convert and go."
- Hype phrases such as "ultimate", "perfect", "magic", "instant", "fast and secure", or "free, bulk, and secure."
- "Not X, but Y" contrast framing.
- Rule-of-three marketing lists.
- Over-personal language such as "we don't think there needs to be."
- Unqualified "no limits"; use "no software-imposed file count limit" and state browser/device constraints.

## Rewrite Examples

| Instead of | Use |
| --- | --- |
| "Privacy-first file tools that run in your browser." | "Browser-native file tools with scoped local processing." |
| "Convert images instantly." | "Convert images in the browser using local device resources." |
| "No limits." | "No software-imposed file count limit; browser memory still applies." |
| "Free, bulk, and secure." | "Batch conversion with local browser processing for supported files." |
| "Runs entirely in your browser." | "Supported file processing runs in your browser." |

## Localization Notes

Localized pages should use the same professional register. Do not translate English hype literally. For CJK and Thai/Vietnamese pages, avoid heavy nominalization, English-calque syntax, and mechanical pronoun repetition when the language-specific research warns against it.
