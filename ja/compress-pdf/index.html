PDF 圧縮の仕組み

ConvertUnlimited は pdf-lib と ブラウザ固有の再エンコード を使用して、PDF ファイルのサイズを削減します。当社のツールは、PDF の内部オブジェクト構造を再構築し、冗長なデータを削除してオブジェクトストリームを最適化しようとします。大きな画像を含む PDF の場合、ブラウザのグラフィックエンジンを使用して視覚データをダウンサンプリングしようとします。

すべての処理はデバイスの CPU と RAM で実行されるため、機密ドキュメントがマシンから離れることはありません。このローカル優先のアプローチにより、100% のプライバシーが確保され、クラウドベースの圧縮サービスに関連するリスクが回避されます。

一部の PDF がほとんど縮小しない理由

PDF 圧縮の結果は、ドキュメントの元の構造によって大きく異なります。理由は以下の通りです：

  • すでに最適化済み： 最近の多くの PDF は、すでに最大圧縮で保存されています。これらの場合、削除できる「余裕」はほとんどありません。
  • テキストのみの PDF： テキストはほとんど容量を使いません。100 ページの PDF が純粋にテキストである場合、そのファイルサイズはすでに最小であり、圧縮の影響はごくわずかです。
  • 暗号化されたファイル： パスワード保護または暗号化された PDF は、セキュリティ上の理由からローカルで再エンコードできないため、サイズを変更することはできません。

テキスト PDF vs スキャンされた PDF

スキャンされた PDF（基本的には一連の画像）が最も圧縮されます。スキャンされたレイヤーの解像度を下げることで、多くの場合、ファイルを 50% 以上縮小できます。Word や Excel から作成されたデジタル PDF は、すでに非常に効率的であり、数パーセント以上縮小することはほとんどありません。

ブラウザベースの PDF 圧縮の制限

クラウドツールは重いサーバー側のアルゴリズムを使用できますが、ブラウザベースのツールはウェブブラウザのメモリと処理制限内で動作する必要があります。ブラウザのクラッシュのリスクなしに、速度、プライバシー、ファイルサイズ削減の最適なバランスを提供する「ベストエフォート型」の構造最適化に焦点を当てています。

よくある質問

機密ドキュメントでも安全ですか？

はい。ドキュメントはブラウザのメモリ内に留まります。当社のサーバーにアップロードされたり、ディスクに保存されたりすることはありません。ページが読み込まれた後は、オフラインでもこのツールを使用できます。

PDF の品質は低下しますか？

「推奨」および「最小ファイル」設定では、特に高解像度のスキャンにおいて、画像の鮮明さがわずかに失われる可能性があります。テキストの鮮明さは通常、オリジナルと同一のままです。

最大ファイルサイズはいくつですか？

ハード制限は設けていませんが、100MB を超えるドキュメントは、再エンコードプロセス中にブラウザに割り当てられたメモリを超える可能性があります。タブがクラッシュした場合は、より強力なデバイスを使用するか、より小さなファイルを試してください。
