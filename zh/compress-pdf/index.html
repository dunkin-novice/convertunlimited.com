PDF 压缩工作原理

ConvertUnlimited 使用 pdf-lib 和 原生浏览器重新编码 来减小 PDF 文件的大小。我们的工具尝试重建 PDF 的内部对象结构，删除冗余数据并优化对象流。对于包含大图像的 PDF，我们尝试使用浏览器的图形引擎对视觉数据进行下采样。

由于所有处理都在您设备的 CPU 和内存上执行，您的敏感文档永远不会离开您的机器。这种本地优先的方法提供 100% 的隐私，并避免了与基于云的压缩服务相关的风险。

为什么有些 PDF 几乎没有缩小

PDF 压缩结果因文档的原始结构而异。原因如下：

  • 已经过优化： 许多现代 PDF 已经以最大压缩率保存。在这些情况下，几乎没有冗余可以删除。
  • 纯文本 PDF： 文本占用的空间非常小。如果您的 100 页 PDF 是纯文本，其文件大小已经很小，压缩影响微乎其微。
  • 加密文件： 出于安全原因，受密码保护或加密的 PDF 无法在本地重新编码，这意味着它们的大小无法更改。

文本 PDF 与扫描 PDF

扫描 PDF（本质上是一系列图像）压缩程度最大。通过降低扫描层的分辨率，我们通常可以将文件缩小 50% 或更多。而从 Word 或 Excel 创建的数字 PDF 已经非常高效，很少能缩小超过几个百分点。

基于浏览器的 PDF 压缩限制

虽然云工具可以使用重型服务器端算法，但基于浏览器的工具必须在 Web 浏览器的内存和处理限制内工作。我们专注于“尽力而为”的结构优化，在速度、隐私和减小文件大小之间提供最佳平衡，而不会导致浏览器崩溃。

常见问题

处理敏感文档安全吗？

是的。您的文档保留在浏览器的内存中。它永远不会上传到我们的服务器或存储在任何磁盘上。页面加载后，您甚至可以离线使用此工具。

我的 PDF 会损失质量吗？

在“推荐”和“小文件”设置下，图像清晰度可能会略有损失，尤其是在高分辨率扫描件上。文本清晰度通常与原始文件完全一致。

最大文件大小是多少？

我们没有设置硬性限制，但超过 100MB 的文档在重新编码过程中可能会超出浏览器分配的内存。如果标签页崩溃，请尝试使用性能更强的设备或更小的文件。
