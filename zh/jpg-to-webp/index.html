ConvertUnlimited 的工作方式

ConvertUnlimited 是一款免费的图片转换工具，完全在你的浏览器中运行。把 JPG、PNG、GIF、WebP、BMP 或 SVG 拖进来，选择输出格式，几秒后就能拿到转换后的副本。没有任何文件被上传到服务器:每张图片都在你本地通过 HTML canvas API 完成解码、重绘和重新编码。

这种本地处理模式有两个实际好处。第一，工具是私密的 —— 文件不离开你的设备，我们(或任何人)都没有什么可以记录、查看或泄露的。第二，我们没有按人头计算的存储成本要承担，所以才能免费、不限量、无需注册、无水印地提供服务。唯一的运营开销是维持网页在线，这部分由右栏的小广告承担。

转换会占用电脑的 CPU 和内存。对大多数笔记本来说，拖入 30–80 张手机相机分辨率的照片几乎是瞬间完成的。非常大的批次 —— 一次几百张高像素图片 —— 会用更多的 RAM，编码期间机器可能短暂变慢。建议从每次约 50 张开始，这是个比较舒服的中间值;但工具不强制上限，你的电脑能扛多少就用多少。批量更大的工作可参考大批量转换技巧。

图片格式指南

不同格式在体积、画质、透明度和兼容性上各有取舍。下面是 ConvertUnlimited 能读取与写出的格式速览，以及在何种情况下选哪种最合适。

WebP

Google 的现代图片格式。在同等画质下，WebP 通常比 JPEG 小 25–35 %、比 PNG 小 50 %，同时支持透明。所有当下浏览器都已支持 WebP，因此它是 Web 场景的最佳默认。当你想要最大化压缩、且受众在浏览器中查看图片时，选 WebP。

JPEG

通用的照片格式。JPEG 是有损格式(为了缩小体积会舍弃部分信息)，不支持透明，但兼容性是真正的「无处不通」—— 邮件、社交、打印机，甚至十年前的旧手机都能打开。当你需要最高的兼容性，或要嵌入到自己无法控制的系统中时，选 JPEG。

PNG

无损，且支持透明。PNG 适合截图、Logo、图标以及任何带有清晰边缘或纯色块的图片，这类内容压缩后不会出现块状失真。代价是体积:照片转 PNG 后通常比同等画质的 JPEG 或 WebP 大数倍。当你需要透明，或画质优先于体积时，选 PNG。

GIF

来自 1990 年代的老格式。GIF 支持基础动画，但用 256 色调色板压缩，因此在照片上看起来块块的。新内容建议静态用 WebP、动画用 MP4 或 WebM。ConvertUnlimited 能读取 GIF，但只转换第一帧 —— 动画 GIF 只会保留首帧。

SVG

由 XML 绘制指令构成的矢量格式。ConvertUnlimited 会读取 SVG 并按其原始尺寸 rasterise(转为像素图)。一旦栅格化就无法再无损放大回矢量，因此只有当你确实需要光栅输出时才转换 SVG —— 比如把 Logo 嵌入 JPEG 产品图。

BMP

未压缩的位图格式。BMP 体积庞大，与所存内容明显不成比例。在今天几乎没有理由继续把图片留作 BMP —— 转成 WebP、JPEG 或 PNG，可以节省 80–95 % 的磁盘空间。

选择合适的画质

画质下拉菜单只对有损格式(WebP、JPEG)生效。PNG 是无损格式，所以选中 PNG 时该菜单会自动禁用。四个预设大致对应:

原始(1.0)。对绝大多数图片来说视觉上与源文件无异。文件仍可能比源文件小 —— 现代 JPEG/WebP 编码器比旧编码器更高效，但你几乎不损失任何细节。

高(0.9)。大多数照片肉眼几乎看不出差异。适合 hero 图、摄影作品，以及对画质有要求的场景。

推荐(0.75)。Web 使用的甜点档。文件比「高」档通常再小 30–50 %，正常观看尺寸下看不出损失。适合博客文章、产品图集、社交媒体。

紧凑(0.5)。较激进的压缩。适合缩略图、图片网格，或要把一文件夹照片用邮件发出去而带宽紧张的场景。细节会有些发软，尤其是文字密集或高对比度的图。

大批量转换技巧

当一次要转的不止几张时，有几点值得注意:

批量大小。建议从每次约 50 张开始，这是个比较舒服的起点。多数笔记本能轻松处理 100–200 张照片大小的图片，但更多就有可能让浏览器在分配内存时短暂卡住。如果出现这种情况，稍等即可 —— 不是崩溃，只是在工作。

RAM 比 CPU 更关键。一张 1200 万像素的照片解码大约要 50 MB RAM。100 张这样的图片，内存峰值瞬间可能去到几 GB。如果撞了天花板，关掉其他标签页通常有帮助。

移动端。手机和平板的 RAM 更少，后台限制也更严。建议在移动设备上保持小批量(10–20 张)。

ZIP 下载。批次转换完成后，「打包 ZIP 下载」按钮会把所有文件打成一个压缩包，比逐个点保存快得多。
