浏览器图片尺寸调整的工作原理

ConvertUnlimited 使用 HTML5 Canvas API 直接在浏览器中调整图片尺寸。当您添加文件时，浏览器会将图片数据解码到 RAM 中。当您指定新尺寸时，浏览器的图形引擎会将该图片绘制到新尺寸的画布上，并应用插值算法以保持图像清晰度。

由于所有计算都在您设备的 CPU 和 GPU 上进行，您的图片永远不会上传到我们的服务器。这种本地优先的方法确保了完全的隐私，并允许近乎即时地处理大批量照片。

调整尺寸 vs 压缩

虽然这两种技术都能减小文件大小，但它们的工作原理不同：

  • 调整尺寸会更改图片的物理尺寸（像素宽度和高度）。这对于满足网站、社交媒体或电子邮件附件的特定要求非常理想。
  • 压缩通过优化颜色数据的存储方式，在保持相同尺寸的同时减小数据大小。

我们的工具允许您通过调整尺寸和输出质量滑块同时完成这两项操作。

理解宽高比

The "锁定宽高比" feature ensures your image doesn't look stretched or squashed. When enabled, changing the width will automatically calculate the correct height based on the original image's proportions (and vice versa). If you need exact specific dimensions that don't match the original shape, uncheck this box, but be aware it may distort the image.

图片尺寸最佳实践

选择合适的尺寸对于网页性能至关重要。以下是一些常用标准：

  • 网站首屏图：宽度 1920px 至 2560px。
  • 博客文章：宽度 800px 至 1200px。
  • 社交媒体帖子：通常为 1080px 至 1200px 的正方形或纵向。
  • 缩略图：宽度 150px 至 300px。

始终尝试将图片调整为它们在屏幕上显示的准确尺寸，以实现最快的加载速度和最佳的 SEO 分数（核心网页指标）。

批量调整尺寸技巧

  • 内存管理：如果您正在调整 100 多张高分辨率照片的尺寸，您的浏览器可能会暂时变慢。为了获得最流畅的体验，请分批处理图片（每批 20-50 张）。
  • ZIP Downloads: After processing a batch, use the "全部下载 (ZIP)" button to bundle all your resized images into one organized file.
  • 格式选择：在现代网站上使用 WebP 可以获得小体积和高质量的最佳平衡。使用 JPG 以获得与旧软件的最大兼容性。

常见问题

有文件大小限制吗？

我们不设软件限制，但非常大的图片（例如 50MB+ 的 RAW 导出）可能会超过浏览器的分配内存。如果标签页崩溃，请尝试一次处理较少的文件。

我的图片会损失质量吗？

缩小尺寸（使图片变小）总是涉及一些数据丢失，但现代浏览器的插值使得这种丢失几乎察觉不到。放大尺寸（使图片变大）可能会导致模糊。

我可以在调整尺寸的同时转换格式吗？

可以。您可以上传 PNG 并直接使用格式选择器将其导出为调整过尺寸的 WebP 或 JPG。
