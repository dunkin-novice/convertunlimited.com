ConvertUnlimited 是如何工作的

ConvertUnlimited 建立在“本地优先”的理念之上与传统的需要您将照片上传到远程服务器的在线图像压缩工具不同，我们的工具直接在您的 Web 浏览器内执行所有压缩任务。

当您将图像拖入压缩器时，您的浏览器会使用 HTML5 Canvas API 来解码图像数据。当您调整质量滑块并点击“压缩”时，浏览器会使用其原生编码引擎将该数据重新编码为您选择的格式（JPG 或 WebP）。由于处理过程发生在您自己的 CPU 和 RAM 上，您的私人照片永远不会离开您的设备。这使得过程更快、更私密，并使我们能够免费提供服务，而无需支付昂贵的服务器端处理费用。

为什么要压缩图像？

未优化的图像通常是导致网站性能缓慢的主要原因。由于以下几个原因，压缩图像是必不可少的：

  • 更快的加载速度： 较小的文件大小意味着更快的下载速度，从而改善访客的用户体验。
  • SEO 优势： 像 Google 这样的搜索引擎将页面速度作为排名因素。更快的网站往往在搜索结果中排名更高。
  • 更好的核心网页指标 (Core Web Vitals)： 减小图像大小有助于改善“最大内容绘制” (LCP)，这是 Google 性能审核中的一个关键指标。
  • 节省存储空间： 如果您处理成千上万张图像，将每张图像缩小 50-80% 可以节省数 GB 的云存储和备份成本。
  • 更容易分享： 压缩后的图像更容易通过电子邮件发送、上传到社交媒体或包含在带宽受限的移动应用中。

压缩格式指南

不同的文件格式使用不同的方法来减小大小。选择正确的格式取决于您的具体需求：

JPEG (Joint Photographic Experts Group)

JPEG 是数码摄影的标准。它使用“有损”压缩，这意味着它会丢弃一些图像数据以实现显著的体积缩减。它具有普遍的兼容性，但不支持透明度。

PNG (Portable Network Graphics)

PNG 使用“无损”压缩，它精确地保留每个像素的原始状态。这使其非常适合徽标、图标和文本密集的图形。虽然它支持透明度，但 PNG 文件通常比 JPEG 或 WebP 大得多。注意：基于浏览器的压缩器通常无法执行“有损” PNG 压缩；如果您需要透明度并希望文件较小，请使用 WebP。

WebP (Google 下一代格式)

WebP 是一种现代格式，同时提供有损和无损压缩。在同等质量下，它的文件大小通常比 JPEG 小 25-35%，并且像 PNG 一样支持透明度。这是现代 Web 性能的最佳选择。

选择压缩质量

质量滑块决定了压缩算法的激进程度：

  • 原始 (100%)： 执行最小的重新编码。主要用于格式转换而不是减小体积。
  • 高 (90-95%)： 在视觉质量几乎没有可察觉损失的情况下显著减小大小。最适合专业的作品集。
  • 推荐 (75-80%)： 大多数 Web 使用的“最佳平衡点”。极大地减小了文件大小，仅在极端放大下才能看到非常轻微的伪影。
  • 紧凑 (50-60%)： 最大限度的压缩。最适合缩略图或速度至关重要且不需要高细节的场景。

批量压缩提示

同时处理几十张图像？请记住以下性能提示：

  • 浏览器内存： 由于压缩发生在您的 RAM 中，处理 100 张以上的超高分辨率图像（如 40MP RAW 导出）可能会导致您的浏览器标签页变慢。对于非常大的文件，请将它们分成较小的批次处理。
  • 移动设备： 手机和平板电脑的可用内存少于台式机。我们建议在移动设备上每批处理 10-20 张图像以获得最佳体验。
  • ZIP 下载： 批量压缩后，使用“全部下载”按钮，通过 JSZip 的支持，在一个结构清晰的单一 ZIP 压缩包中获取所有文件。

常见问题

它真的很私密吗？

是的。打开浏览器开发者工具中的网络检查器，您会看到没有任何图像数据被发送到我们的服务器。所有的逻辑都在您的浏览器本地环境中执行。

我的文件会被上传吗？

永远不会。我们的服务器提供的只是在您的浏览器中运行该工具所需的静态代码。

为什么它是免费的？

因为我们不需要支付昂贵的“GPU 加速”服务器来处理您的图像。您的电脑完成了这项工作，这使我们的运营成本低至可以被简单的静态广告所覆盖。

压缩会降低质量吗？

在有损格式（JPG/WebP）中，是的，但这是一个数学上的权衡。在 80% 的质量下，标准显示器上的人眼通常无法察觉这种“损失”。

为什么我的 PNG 有时会变大？

如果您将高度压缩的 JPG 转换为 PNG，文件大小会增加，因为 PNG 是一种旨在保留数据而不是丢弃数据的无损格式。
